# TFP v3.2.1 Identity Security Release

## Overview
v3.2.1 implements **encrypted identity storage** with backup and recovery capabilities, addressing the CRITICAL security vulnerability where plaintext `~/.tfp/identity.json` files could lead to permanent credit loss or account takeover.

## Changes

### New Files
- `tfp_cli/identity.py` - Core identity management module with:
  - AES-256-GCM encryption (PBKDF2 key derivation, 100k iterations)
  - Mnemonic-based recovery (12-word phrase)
  - Automatic backup on creation/modification
  - Export functionality for offline backups
  - Passphrase change capability

### Modified Files
- `tfp_cli/main.py` - Updated all commands to use encrypted identity:
  - `publish`, `earn`, `join` now require passphrase
  - New `identity` subcommand with `export`, `recover`, `change-passphrase`
  - Environment variable support: `TFP_IDENTITY_PASSPHRASE`

### Tests Updated
- `tests/test_tfp_cli.py` - Fixed to work with encrypted identity

## Usage

### First-Time Setup
```bash
# Set passphrase (or use --passphrase flag)
export TFP_IDENTITY_PASSPHRASE="your-secure-passphrase"

# Join compute pool - will generate mnemonic
python -m tfp_cli.main join --device-id my-device

# ⚠️ WRITE DOWN THE MNEMONIC - it's your only recovery option!
```

### Backup Identity
```bash
python -m tfp_cli.main identity export --passphrase "your-passphrase"
# Outputs: ~/.tfp/backups/export_TIMESTAMP.enc
```

### Recover from Mnemonic
```bash
python -m tfp_cli.main identity recover \
  --mnemonic "alpha bravo charlie ..." \
  --device-id my-device \
  --passphrase "new-passphrase"

# ⚠️ Credits from old identity are NOT recoverable (by design)
```

### Change Passphrase
```bash
python -m tfp_cli.main identity change-passphrase \
  --old-passphrase "old" \
  --new-passphrase "new"
```

## Security Properties

### What's Protected
✅ Identity file encrypted at rest (AES-256-GCM)  
✅ Key derived from passphrase (PBKDF2-SHA256, 100k iterations)  
✅ File permissions set to 0600 (owner read/write only)  
✅ Automatic backups created (last 5 retained)  
✅ Mnemonic recovery available  

### Important Warnings
⚠️ **Lost passphrase = permanent identity loss** (no backdoor)  
⚠️ **Lost mnemonic = cannot recover from file corruption**  
⚠️ **Credits bound to PUF entropy - NOT recoverable via mnemonic**  
⚠️ **Mnemonic recovery creates NEW identity (same device ID, different PUF)**  

## Migration from Plaintext

Existing `~/.tfp/identity.json` files are **NOT** automatically migrated. To migrate:

1. Note your current device ID
2. Run any command with new code - it will create encrypted identity
3. Re-enroll device with server (credits remain intact on server side)
