## 3. API Reference

All endpoints are served by the FastAPI demo node (`uvicorn tfp_demo.server:app`).
Interactive docs at `http://localhost:8000/docs`.

### `GET /health`
Quick liveness check.
```json
{"status": "ok", "content_items": 3}
```

### `GET /api/status`
Extended node status including Nostr subscriber state.
```json
{
  "version": "0.2.0",
  "content_items": 5,
  "nostr_events_received": 12,
  "nostr_subscriber_running": true,
  "nostr_relay": "wss://relay.damus.io"
}
```

### `POST /api/enroll`
Register a device's PUF entropy. **Must be called before `publish` or `earn`.**
```json
// Request
{ "device_id": "my-phone-001", "puf_entropy_hex": "<64-hex-chars>" }
// Response
{ "enrolled": true, "device_id": "my-phone-001" }
```

### `POST /api/earn`
Earn 10 credits by submitting a compute task.
Requires `X-Device-Sig: <HMAC-SHA-256(puf_entropy, "device_id:task_id")>` header.
```json
// Request body
{ "device_id": "my-phone-001", "task_id": "task-abc-001" }
// Response
{ "device_id": "my-phone-001", "task_id": "task-abc-001", "credits_earned": 10, "chain_hash": "..." }
```
Errors: `401` (unknown/bad sig), `422` (missing header).

### `POST /api/publish`
Publish content. Requires valid `X-Device-Sig` header.
Signature covers `"device_id:title"`.
```json
// Request body
{
  "device_id": "my-phone-001",
  "title": "Radio Show S01E01",
  "text": "Full episode transcript ...",
  "tags": ["radio", "season1"]
}
// Response
{ "root_hash": "<sha3-256-hex>", "title": "...", "tags": [...], "status": "broadcasting" }
```

### `GET /api/content?tag=<tag>`
List all content, optionally filtered by tag (exact match, lowercase normalized).
```json
{ "items": [{ "root_hash": "...", "title": "...", "tags": [...] }] }
```

### `GET /api/get/{root_hash}?device_id=<id>`
Retrieve content. Requires prior `earn` call for `device_id` (spends 1 credit).
```json
{ "root_hash": "...", "title": "...", "tags": [...], "text": "...", "sha3": "..." }
```
Errors: `402` (no credits), `404` (hash unknown).

### `GET /api/discovery?domain=<domain>`
Return content hashes discovered via Nostr announcements (remote peers).
```json
{
  "domain": "general",
  "entries": [{"content_hash": "...", "tag": "radio", "popularity": 0.5}],
  "source": "nostr"
}
```

### Static PWA assets
| Path | Description |
|------|-------------|
| `GET /` | Demo SPA (index.html) |
| `GET /manifest.json` | PWA manifest (installable) |
| `GET /service-worker.js` | Offline service worker |

## 5. Device Auth & Enrollment

### Design
Each device generates 32 bytes of entropy once (simulating PUF — Physical Unclonable Function). This entropy is stored server-side as the device's shared secret.

Every mutating API call signs its key fields with HMAC-SHA-256:

```
X-Device-Sig = HMAC-SHA-256(puf_entropy, "device_id:title")   # for /api/publish
X-Device-Sig = HMAC-SHA-256(puf_entropy, "device_id:task_id") # for /api/earn
```

### Enrollment flow
```python
import secrets, hmac, hashlib

puf_entropy = secrets.token_bytes(32)
device_id = "device-" + puf_entropy.hex()[:12]

# 1. Enroll
requests.post("/api/enroll", json={
    "device_id": device_id,
    "puf_entropy_hex": puf_entropy.hex()
})

# 2. Sign a request
message = f"{device_id}:{task_id}"
sig = hmac.new(puf_entropy, message.encode(), hashlib.sha256).hexdigest()

# 3. Earn
requests.post("/api/earn",
    json={"device_id": device_id, "task_id": task_id},
    headers={"X-Device-Sig": sig}
)
```

### Security properties
- Server uses `hmac.compare_digest` (constant-time) — no timing oracle.
- Missing header → FastAPI 422 (validation error, not leaking info).
- Unknown device or wrong entropy → 401.
- Replay protection: different `task_id` per earn call (client responsibility).

### Browser (SubtleCrypto)
The PWA demo (`demo/index.html`) uses `crypto.subtle.importKey` + `crypto.subtle.sign("HMAC", ...)` with the stored PUF entropy — no server-side JS frameworks required.

## 6. SQLite Persistence

### Schema
```sql
-- content table
CREATE TABLE content (
    root_hash TEXT PRIMARY KEY,
    title     TEXT NOT NULL,
    tags      TEXT NOT NULL,   -- JSON array e.g. ["demo","radio"]
    data      BLOB NOT NULL
);

-- devices table
CREATE TABLE devices (
    device_id   TEXT PRIMARY KEY,
    puf_entropy BLOB NOT NULL,
    enrolled_at REAL NOT NULL  -- Unix timestamp
);
```

### In-memory tag cache
`ContentStore` maintains `_tag_index: dict[str, set[str]]` (tag → root_hashes).

| Operation | Complexity |
|-----------|------------|
| `put(item)` | O(T) — T tags; SQLite write + cache update |
| `get(root_hash)` | O(1) SQLite rowid lookup |
| `filter_by_tag(tag)` | O(M) — M matches; IN query by hash set |
| `count()` | O(1) SQLite COUNT |
| `all()` | O(N) — full scan |

The cache is rebuilt from DB on every `ContentStore.__init__` call (i.e., on server startup).

### DB file location
| Env var | Default |
|---------|---------|
| `TFP_DB_PATH` | `<repo>/tfp-foundation-protocol/pib.db` |
| `TFP_DB_PATH=:memory:` | In-memory (tests, ephemeral) |

## 7. Nostr Bridge & Subscriber

### Round-trip
```
Local node                Nostr Relay           Remote peer
──────────                ──────────            ──────────
NostrBridge.publish()──►  EVENT(kind=30078)  ◄──(announces content)
                              │
NostrSubscriber─────────►  REQ filter
     │
     ▼ on_event callback
TagOverlayIndex.add_entry()
```

### NostrBridge (publisher)
```python
from tfp_client.lib.bridges.nostr_bridge import NostrBridge

bridge = NostrBridge(privkey=my_32_byte_key, relay_url="wss://relay.damus.io")
event = bridge.publish_content_announcement(
    content_hash="abc...64hex",
    metadata={"title": "My Show", "tags": ["radio"], "domain": "entertainment"}
)
print(event.id)    # NIP-01 event id
print(event.sig)   # Schnorr signature
```

Offline (no relay):
```python
bridge = NostrBridge(offline=True)
event = bridge.build_content_announcement(content_hash, metadata)
```

### NostrSubscriber (receiver)
```python
from tfp_client.lib.bridges.nostr_subscriber import NostrSubscriber

def on_tfp_event(event_dict):
    import json
    payload = json.loads(event_dict["content"])
    print("Discovered:", payload["hash"])

sub = NostrSubscriber(
    relay_url="wss://relay.damus.io",
    on_event=on_tfp_event,
    reconnect_delay=5.0,   # seconds between reconnect attempts
    offline=False,
)
sub.start()   # daemon thread

# ... later ...
sub.stop()
events = sub.get_received()  # list of all received NIP-01 event dicts
```

### Demo server wiring
The server lifespan starts a subscriber automatically. Set `NOSTR_RELAY=wss://...` to connect to a real relay. Without it, the subscriber thread exits immediately (offline mode) but `/api/discovery` and `_on_nostr_event` are still wired up for test injection.

## 8. PWA & Service Worker

### Installation
1. Open `http://localhost:8000` in Chrome/Edge/Safari.
2. Browser shows "Install" prompt (Android: "Add to Home Screen", Desktop: address bar install icon).
3. App launches standalone (no browser chrome).

### Caching strategy
| URL Pattern | Strategy | Rationale |
|-------------|----------|-----------|
| `/`, `/manifest.json` | Cache-first | Static; change only on deploy |
| `/api/get/*` | Network-first, cache on success | Content playback works offline |
| `/api/content` | Cache-first | Discovery works offline |
| `/api/publish`, `/api/earn`, `/api/enroll` | Network-only | Mutations must reach server |

### Device identity
The demo generates a random 32-byte entropy on first load, stores it in `localStorage`, and re-uses it for all sessions on that browser. This simulates a PUF-bound device identity.

### Updating the service worker
Increment `CACHE_NAME` in `demo/service-worker.js` when static assets change. The `activate` handler automatically clears old caches.

## 9. Gateway Demand Scheduling

### `schedule_from_aggregator` (new direct path)

```python
from tfp_client.lib.publish.mesh_aggregator import MeshAggregator
from tfp_broadcaster.src.gateway.scheduler import GatewayScheduler

# Aggregator collects live demand
agg = MeshAggregator(region="us-west")
agg.receive_announcement("abc...64", {"title": "Popular Show"})
agg.increment_demand("abc...64", count=80)

# Scheduler reads live demand and schedules
sched = GatewayScheduler(max_slots_per_epoch=10)
epoch = int(time.time() // 3600)  # current hour
slots = sched.schedule_from_aggregator(agg, epoch)

for slot in slots:
    print(f"Slot {slot.slot_id}: {slot.content_hash[:8]}… bid={slot.bid_amount}")

# After broadcasting
sched.mark_broadcasted(epoch, slots[0].slot_id)
```

**Key behaviors:**
- Slots are ordered by `demand_score` descending (highest demand first).
- After scheduling, `aggregator.reset_demand(hash)` is called for each scheduled item — prevents double-counting in the next epoch.
- Unscheduled items (didn't fit) retain their demand and will be considered in the next epoch.
- Legacy path (`receive_aggregated_demand` + `schedule_from_demand`) still works and produces equivalent results.

### Bid formula
```
base_bid = demand_score × base_credit_rate
size_penalty = content_size_bytes / 1_000_000
load_multiplier = 1.0 / (1.0 + current_load)
final_bid = max(1, base_bid × (1 + size_penalty) × load_multiplier)
```

## 12. Extension Guide

### Adding a new protected endpoint

1. Add a `device_id` field to the request model.
2. Define the HMAC message string (must be unique to the operation).
3. Add `x_device_sig: str = Header(alias="X-Device-Sig")` to the handler.
4. Call `_verify_device_sig(device_id, x_device_sig, message, _device_registry)`.

Example:
```python
@app.post("/api/my_action")
def my_action(
    payload: MyRequest,
    x_device_sig: str = Header(alias="X-Device-Sig"),
) -> dict:
    message = f"{payload.device_id}:my_action:{payload.some_field}"
    if not _verify_device_sig(payload.device_id, x_device_sig, message, _device_registry):
        raise HTTPException(status_code=401, detail="invalid device signature")
    # ... your logic ...
```

### Persisting new data to SQLite

Add a new manager class following the `ContentStore` pattern:

```python
class MyStore:
    def __init__(self, conn: sqlite3.Connection) -> None:
        self._conn = conn
        self._conn.execute("CREATE TABLE IF NOT EXISTS my_table (...)")
        self._conn.commit()
```

Wire it in the `lifespan` context manager alongside `ContentStore` and `DeviceRegistry`.

### Handling Nostr events beyond content announcements

Override `_on_nostr_event` or pass a custom callback to `NostrSubscriber`. The raw NIP-01 event dict is delivered; `event["content"]` is a JSON string with TFP-specific payload.

```python
def my_callback(event: dict):
    payload = json.loads(event["content"])
    # payload keys: hash, title, tags, domain
    my_index.register(payload["hash"], payload.get("tags", []))

sub = NostrSubscriber(relay_url="wss://...", on_event=my_callback)
sub.start()
```

## 13. Threat Model Updates

### New threats addressed in v2.12

| Threat | Mitigation |
|--------|-----------|
| **Unauthenticated spam** | Device enrollment + HMAC-SHA-256 per-request signature required for all mutating endpoints |
| **Credit duplication via replay** | Unique `task_id` per earn call; server does not deduplicate (client responsibility — future: store seen task_ids in DB) |
| **Data loss on restart** | SQLite persistence — all content and device enrollments survive process restarts |
| **Nostr event injection** | Events parsed defensively; exceptions in `on_event` are caught and logged, never propagated |
| **Tag query amplification** | In-memory tag index prevents repeated full-table scans from becoming DoS vectors |

### Remaining open items

| Item | Priority | Notes |
|------|----------|-------|
| Task-ID deduplication | Medium | Store seen task IDs in SQLite to prevent credit replay within a session |
| Rate limiting | Medium | Add per-device rate limits on `/api/earn` to prevent rapid credit farming |
| Nostr event signature verification | Low | Currently trusts relay filtering; add BIP-340 Schnorr verification of received events |
| DB migration | Low | Add schema version table for future schema upgrades without data loss |
