# TFP v3.2 Production Deployment Checklist

**Version:** 3.2.0  
**Last Updated:** 2024  
**Prerequisites:** Docker, Docker Compose, domain name (for HTTPS)

---

## Pre-Deployment

### 1. Identity Security (CRITICAL)
- [ ] Generate strong passphrase (16+ characters)
- [ ] Set `TFP_IDENTITY_PASSPHRASE` environment variable
- [ ] Run initial enrollment to create encrypted identity
- [ ] **BACKUP MNEMONIC PHRASE** - Write to paper/metal, store offline
- [ ] Verify recovery works: `tfp identity recover --mnemonic "your-phrase"`
- [ ] Test encrypted backup/restore cycle

### 2. Rate Limiting Configuration
- [ ] Deploy Redis server (included in docker-compose.yml)
- [ ] Set `REDIS_URL=redis://redis:6379/0` in environment
- [ ] Test rate limiting under load (30+ requests/minute)
- [ ] Verify limits persist after server restart

### 3. Database Setup
- [ ] Choose storage backend:
  - SQLite: `<100 concurrent users`, set `TFP_DB_PATH=/data/tfp.db`
  - PostgreSQL: `>100 users`, uncomment postgres in docker-compose.yml
- [ ] Configure persistent volume for database
- [ ] Test database backup/restore procedure
- [ ] Enable WAL mode (default in v3.2)

### 4. Network & TLS
- [ ] Obtain domain name
- [ ] Update `deploy/Caddyfile` with your domain
- [ ] Configure DNS A record to server IP
- [ ] Test HTTP → HTTPS redirect
- [ ] Verify SSL certificate auto-renewal

### 5. Monitoring Setup
- [ ] Enable OpenTelemetry metrics
- [ ] Configure Prometheus scraper (port 8000)
- [ ] Set up Grafana dashboard (optional)
- [ ] Create alert rules:
  - [ ] HABP consensus failure rate > 5%
  - [ ] Rate limit hits > 100/hour
  - [ ] Credit ledger integrity check failures
  - [ ] Disk usage > 80%

## Deployment Steps

### Step 1: Prepare Environment
```bash
cd tfp-foundation-protocol/deploy

# Create .env file with production values
cat > .env << ENV
TFP_IDENTITY_PASSPHRASE=your-strong-passphrase-here
REDIS_URL=redis://redis:6379/0
TFP_DB_PATH=/data/tfp.db
TFP_ENV=production
NOSTR_RELAY=wss://relay.damus.io
ENV

# Update Caddyfile with your domain
sed -i 's/localhost/your-domain.com/' Caddyfile
```

### Step 2: Launch Services
```bash
docker compose up -d

# Wait for services to be healthy
docker compose ps
# All services should show "healthy"

# Check logs for errors
docker compose logs tfp-server | grep -i error
```

### Step 3: Verify Deployment
```bash
# Health check
curl https://your-domain.com/health

# Should return: {"status":"ok"}

# Metrics endpoint
curl https://your-domain.com/metrics

# Should return Prometheus-format metrics

# Enroll first device
export TFP_IDENTITY_PASSPHRASE="your-passphrase"
python -m tfp_cli.main --api https://your-domain.com join
```

### Step 4: Consensus Testing (CRITICAL)
⚠️ **Remember: Single nodes cannot earn credits**

```bash
# Start 3+ devices for HABP consensus
# Terminal 1
python -m tfp_cli.main --api https://your-domain.com join --device-id device-1 &

# Terminal 2
python -m tfp_cli.main --api https://your-domain.com join --device-id device-2 &

# Terminal 3
python -m tfp_cli.main --api https://your-domain.com join --device-id device-3 &

# Monitor consensus in server logs
docker compose logs -f tfp-server | grep "HABP consensus"
```

## Troubleshooting

### Issue: "Connection refused" to Redis
```bash
# Check Redis is running
docker compose ps redis

# View Redis logs
docker compose logs redis

# Restart if needed
docker compose restart redis
```

### Issue: Identity file corrupted
```bash
# Restore from mnemonic
export TFP_IDENTITY_PASSPHRASE="new-passphrase"
tfp identity recover --mnemonic "your-backup-phrase"

# Verify credits restored
python -m tfp_cli.main status
```

### Issue: HABP consensus not reaching
```bash
# Verify 3+ devices connected
docker compose logs tfp-server | grep "device enrolled"

# Check network connectivity
docker compose exec tfp-server curl http://redis:6379

# Review consensus threshold
# Default: 3/5 devices must agree
```

### Issue: HTTPS not working
```bash
# Check Caddy logs
docker compose logs caddy

# Verify domain DNS
dig your-domain.com

# Check port 80/443 open
nmap -p 80,443 your-domain.com
```

### Issue: Database corruption after crash
```bash
# SQLite WAL recovery (automatic)
docker compose restart tfp-server

# PostgreSQL recovery
docker compose logs postgres | grep "recovery"

# Restore from backup if needed
# See Backup & Recovery section
```

## Backup & Recovery

### Automated Backups
```bash
# Add to crontab (daily at 2 AM)
0 2 * * * docker compose exec tfp-server cp /data/tfp.db /backups/tfp-$(date +\%F).db

# Identity backup (manual, secure location)
cp ~/.tfp/identity.enc /secure-offline-storage/
cp ~/.tfp/mnemonic.txt /secure-offline-storage/
```

### Disaster Recovery
1. Provision new server
2. Install Docker/Docker Compose
3. Copy backup files to new server
4. Restore identity: `tfp identity recover --mnemonic`
5. Restore database from backup
6. Update DNS if IP changed
7. Verify all services healthy

## Security Hardening

### Firewall Rules (Minimum)
```bash
# Allow HTTP/HTTPS
ufw allow 80/tcp
ufw allow 443/tcp

# Allow SSH (restrict to known IPs)
ufw allow from <admin-ip> to any port 22

# Block direct access to 8000 (only via Caddy)
ufw deny 8000/tcp

# Enable firewall
ufw enable
```

### Security Scans
```bash
# Run bandit security scan
bandit -r tfp_client/ tfp_demo/

# Run semgrep
semgrep --config=auto .

# Check for known vulnerabilities
safety check -r requirements.txt
```

### Access Control
- [ ] Restrict SSH to key-based auth only
- [ ] Enable fail2ban for brute-force protection
- [ ] Use separate user for TFP services (not root)
- [ ] Rotate passphrases quarterly
- [ ] Audit access logs monthly

## Performance Tuning

### For >100 Concurrent Users
1. Switch to PostgreSQL (uncomment in docker-compose.yml)
2. Increase Redis memory: `command: redis-server --maxmemory 256mb`
3. Add connection pooling in application
4. Consider read replicas for content queries

### For High Task Volume
1. Increase HABP redundancy factor (default: 5)
2. Scale compute workers horizontally
3. Use faster task types (hash_preimage vs matrix_verify)
4. Monitor task queue depth
