# TFP v2.8: Security Hardening & Edge Case Analysis

## 🚨 Critical Threat Landscape

You identified two **existential threats** that standard checksums miss:
1.  **Steganographic/Malicious Payload Injection**: A popular video contains hidden malware in unused DCT coefficients or metadata, bypassing hash checks because the *file* is valid, but the *content* is toxic.
2.  **Supply Chain Poisoning (RAT Injection)**: An attacker compromises a plugin update or a compute node to inject Remote Access Trojans (RATs) into the execution environment.

### 🔍 Blind Spot Analysis: What We Missed

| Threat Vector | Current Defense | Gap | Risk Level |
| :--- | :--- | :--- | :--- |
| **Semantic Malware** (Hidden payload in valid media) | HMAC-RC (Integrity) | No deep inspection of *decoded* content | 🔴 Critical |
| **Plugin Supply Chain** (Malicious update) | Signature Check | No sandboxing or behavioral analysis | 🔴 Critical |
| **Compute Node Exfiltration** (Task steals data) | TEE Attestation | TEEs can have side-channel leaks; no network egress control | 🟠 High |
| **Sybil Reputation Farming** (Fake popularity) | PUF Identity | PUF proves hardware uniqueness, not honest behavior | 🟡 Medium |
| **Gradient Leakage** (AI Model inversion) | Encryption | No differential privacy on intermediate compute steps | 🟡 Medium |
| **Resource Exhaustion** (DoS via complex tasks) | Device Safety | Guards protect hardware, not network bandwidth from spam tasks | 🟡 Medium |

## 🛡️ v2.8 Solution Architecture: "Zero-Trust Execution"

We add **3 Defensive Layers** without breaking the "Uncensorable" promise. The goal is not to censor *hashes*, but to make executing malicious content **economically suicidal** and **technically contained**.

### Module 1: Semantic Sandboxing (`tfp_core/security/sandbox.py`)
**Problem**: Malicious video executes code on the player.
**Solution**: WebAssembly (Wasm) sandbox for all content rendering and plugin execution.
- **Mechanism**: All plugins and media decoders run in a headless Wasm runtime (e.g., `wasmer` or `wasmtime`).
- **Capabilities**: Zero host filesystem access, zero network access (unless explicitly granted by user prompt).
- **Outcome**: Even if a video contains a RAT, it runs in a void. It cannot phone home or touch your files.

### Module 2: Probabilistic Content Scanning (`tfp_core/security/scanner.py`)
**Problem**: Hidden malware in popular content.
**Solution**: Demand-triggered heuristic scanning.
- **Mechanism**: 
    - When content crosses a popularity threshold (e.g., >100 requests), it triggers a **"Community Audit"**.
    - Randomly selected high-reputation nodes (verified via PUF + History) download and decode the content in their sandbox.
    - They run lightweight heuristics (entropy checks for steganography, signature scans for known blobs).
    - **Consensus**: If >60% of auditors flag it, the content gets a **"Toxic" metadata tag**.
- **Economic Incentive**: Auditors earn **2x Credits**. If they lie (false positive on clean content), they lose their reputation stake.

### Module 3: Network Egress Control (`tfp_core/compute/task_mesh.py` update)
**Problem**: Compute task exfiltrates user data.
**Solution**: Strict egress policies per task type.
- **Mechanism**: 
    - `RENDER` tasks: Network access = **DENY**. Output = **ONLY** specific buffer.
    - `INDEX` tasks: Network access = **READ-ONLY** (input hash).
    - `RELAY` tasks: Network access = **PROXY ONLY** (cannot initiate new connections).
- **Enforcement**: Enforced by the Sandbox and the Host OS firewall rules generated by the TFP client.

## ⚖️ The "Uncensorable" Paradox Resolution

**Q:** Doesn't scanning violate "Uncensorable"?
**A:** No. 
1.  **The Hash Still Resolves**: The content is still downloadable. We do not block the hash.
2.  **User Choice**: The "Toxic" tag is just metadata. A user *can* choose to ignore it and view the content (e.g., for research).
3.  **Default Safety**: The *default* renderer refuses to execute code flagged as toxic, protecting the average user.
4.  **Decentralized Judgment**: The "Toxic" label comes from random peers, not a central authority. To censor, you'd need to compromise 51% of the global audit pool simultaneously.

## 📝 Code Structure Preview

```python
# tfp_core/security/sandbox.py
class SecureSandbox:
    def __init__(self, capabilities: List[Capability]):
        self.wasm_runtime = wasmer.Store()
        self.capabilities = capabilities # e.g., [NETWORK_READ, FS_WRITE_TEMP]
        
    def execute_plugin(self, plugin_bytes: bytes, input_data: bytes) -> bytes:
        # 1. Load into isolated memory
        instance = self.wasm_runtime.create_instance(plugin_bytes)
        
        # 2. Trap syscalls (open, socket, exec) unless in capabilities
        instance.import_object("env", self._secure_host_functions())
        
        # 3. Run with timeout (prevent infinite loops)
        try:
            return instance.exports.run(input_data, timeout_ms=5000)
        except wasmer.RuntimeError as e:
            raise SecurityViolation(f"Sandbox escape attempt: {e}")

# tfp_core/security/scanner.py
class CommunityAuditor:
    def audit_content(self, content_hash: str) -> AuditResult:
        # 1. Fetch content
        data = ndn.fetch(content_hash)
        
        # 2. Decode in sandbox
        decoded = SecureSandbox([]).decode_media(data)
        
        # 3. Heuristics
        if self._check_steganography(decoded): return AuditResult.FLAGGED
        if self._check_known_signatures(decoded): return AuditResult.FLAGGED
        
        return AuditResult.CLEAN

```
